# FileShare 使用方法

最終更新: 2026-06-26 / バージョン: 1.0.0

このドキュメントでは、FileShare プラグインの使い方を管理者向け・利用者向けに説明します。

---

## 1. 事前準備（インストールと有効化）

1. `fileshare` フォルダを `wp-content/plugins/` に配置します。
2. WordPress 管理画面 →「プラグイン」→ **FileShare** を有効化します。
3. 有効化時に以下が自動で行われます。
   - 動作要件（WP 6.2+ / PHP 8.1+ / sqlite3 / zip）のチェック
   - メタデータ用テーブルの作成
   - 保存ディレクトリ `wp-content/uploads/fileshare-files/` の作成と直アクセス遮断
   - SQLite ログ DB の初期化
   - 自動クリーンアップ（毎時）の登録

---

## 2. 管理者の操作

### 2-1. ファイルをアップロードする

1. 管理メニュー **FileShare** を開きます。
2. 「ファイルをアップロード」エリアに、ファイルを**ドラッグ&ドロップ**するか、クリックして選択します。
   - 複数ファイルを選ぶと、自動的に 1 つの ZIP に圧縮されます。
3. 必要に応じて以下のオプションを設定します。

   | 項目 | 説明 |
   |------|------|
   | パスワード | 空欄の場合は自動生成されます。 |
   | 最大ダウンロード回数 | `0` で無制限。指定回数に達すると自動的にダウンロード不可になります。 |
   | 保存期間（日数） | `0` で無期限。経過後は自動削除されます（既定 7 日）。 |

4. 「アップロード」をクリックします。
5. 完了後、**ダウンロード ID / パスワード / ダウンロードページ URL** が表示されます。
   ダウンロードページ URL は専用URL（`?fileshare_download=1&fsid=XXXX`）で、固定ページを作成しなくてもそのまま相手に共有できます。

> ⚠️ **パスワードはこの画面でのみ表示されます。** 必ずこの時点で控えてください。
> 各項目の「コピー」ボタンでクリップボードにコピーできます。

### 2-2. 相手に共有する

利用者には以下を伝えます。

- ダウンロードページの URL（専用URL `?fileshare_download=1&fsid=XXXX`・固定ページ不要）
- ダウンロード ID
- パスワード

### 2-3. 保存済みファイルを管理する

同じ FileShare 画面の「保存済みファイル」一覧で、各ファイルの
ファイル名 / ID / サイズ / ダウンロード状況 / 期限 を確認できます。
「削除」ボタンで実ファイルとメタデータを即時削除できます。

### 2-4. アクセスログを確認する

管理メニュー **FileShare → アクセスログ** で、
アップロード・ログイン試行・ダウンロード・ブロック・レート制限の記録を確認できます（直近 200 件）。
※ このページの閲覧には `manage_options` 権限が必要です。

### 2-5. フロントアップロードを設定する（社内向け）

管理画面に入らず、専用URLからアップロードさせたい場合に使用します。
WordPress のユーザーアカウントとは別に、**専用の ID/パスワードを 1 組**設定します。
**固定ページやショートコードの設置は不要です。**

1. 管理メニュー **FileShare → フロント設定** を開きます（`manage_options` 権限が必要）。
2. 「ユーザー名」「パスワード」を入力します。
   - パスワードは `password_hash()` でハッシュ化して保存され、画面上には表示されません。
   - 設定済みのパスワードを変えない場合は、パスワード欄を空欄のまま保存します。
3. 「フロントアップロードを有効にする」にチェックを入れて保存します。
4. 保存すると、同じ画面の「専用アップロードURL」欄にURL（例: `https://example.com/?fileshare_upload=1`）が表示されます。これを担当者に共有します。

> 💡 ユーザー名を変更すると、既存のログインセッションはすべて無効になります（再ログインが必要）。
> 専用URLにアクセスすると専用ログイン画面が表示され、未ログインの状態ではアップロードフォームは表示されません。

---

## 3. ダウンロードページ（固定ページ不要）

利用者のダウンロード窓口は**専用URL**で提供されます。固定ページやショートコードの設置は不要です。

- ダウンロードページの専用URL: `https://example.com/?fileshare_download=1`
- ファイルをアップロードすると、管理画面の「アップロード結果」にダウンロード ID 入りの専用URL
  （例: `https://example.com/?fileshare_download=1&fsid=abcd1234ef56`）が表示されます。
  この URL をそのまま相手に共有してください。

### URL にダウンロード ID を埋め込む

- URL パラメータ `fsid` で ID を渡すと、ダウンロードフォームに自動入力されます。

  ```
  https://example.com/?fileshare_download=1&fsid=abcd1234ef56
  ```

- `fsid` を付けない専用URL（`?fileshare_download=1`）を開くと、ID とパスワードの両方を手入力する
  フォームが表示されます。

---

## 4. 利用者の操作（ダウンロード）

1. 共有されたダウンロードページ URL を開きます。
2. **ダウンロード ID** と **パスワード** を入力します。
3. 「認証してダウンロード」をクリックします。
4. 認証に成功すると「ダウンロード」ボタンが表示され、ファイルを取得できます。

### うまくいかない場合

| 症状 | 原因 / 対処 |
|------|------------|
| ダウンロードページ URL を開いても**フォームが表示されない** | 専用URL（`?fileshare_download=1` を含む URL）を開いているか確認してください。アップロード結果画面に表示される URL をそのまま共有していれば、固定ページを作成しなくてもフォームが表示されます。サイトのトップ URL（`?fileshare_download=1` が無い URL）を開いている場合は表示されません。 |
| 「ID またはパスワードが正しくありません」 | 入力内容を確認してください。連続失敗するとブロックされます。 |
| 「試行回数が多すぎます…」 | ブルートフォース対策で一時的に IP がブロックされています。時間をおいて再試行してください。 |
| 「保存期限が切れています」 | 保存期間を過ぎたため削除されています。再アップロードが必要です。 |
| 「ダウンロード回数の上限に達しています」 | 最大回数に達しています。再アップロードが必要です。 |
| 「アクセスが集中しています…」 | レート制限中です。少し待って再試行してください。 |

---

## 5. 利用者の操作（フロントアップロード）

社内の担当者が専用URLからアップロードする手順です（事前に 2-5 の設定が必要）。

1. 管理者から共有された**専用アップロードURL**（例: `https://example.com/?fileshare_upload=1`）を開きます。
2. 管理者から共有された**ユーザー名**と**パスワード**でログインします。
3. ログイン後、アップロードフォームが表示されます。
   - ファイルを選択（複数選択時は自動的に ZIP 圧縮）。
   - 必要に応じてパスワード（空欄なら自動生成）・最大ダウンロード回数・保存期間を設定。
4. 「アップロード」をクリックすると、**ダウンロード ID / パスワード / ダウンロードページ URL** が表示されます。
   （この情報は表示された画面でのみ確認できます。控えてから相手に共有してください。）
5. 作業が終わったら「ログアウト」してください。ログイン状態は最大 1 時間で自動的に失効します。

> 連続してログインに失敗すると、ダウンロード認証と同様に IP が一時ブロックされます。

---

## 6. 運用上の注意

- **Nginx 環境**では `.htaccess` が無効です。`wp-content/uploads/fileshare-files/` への直接アクセスを
  Web サーバ設定（`deny all` 相当）で必ず遮断してください。
- 自動クリーンアップは WordPress の Cron に依存します。アクセスが極端に少ないサイトでは
  実行が遅延することがあります（必要に応じて実 cron（WP-CLI / システム cron）の併用を検討してください）。
- プラグインを**削除**すると、保存ファイル・ログ・テーブルはすべて消去されます。

## 7. 既定のセキュリティパラメータ

| 項目 | 既定値 |
|------|--------|
| ダウンロード認証トークン有効期間 | 600 秒 |
| フロントログインのセッション有効期間 | 3600 秒（1 時間） |
| IP ブロック判定ウィンドウ | 900 秒（15 分） |
| ブロック発動しきい値 | 5 回失敗 |
| ブロック継続時間 | 1800 秒（30 分） |
| ダウンロード認証レート制限 | 1 分あたり 10 回 |
| フロントログインレート制限 | 1 分あたり 10 回 |
| フロントアップロードレート制限 | 1 分あたり 20 回 |
| ログ保持期間 | 30 日 |
